第一部分：導言 (Introduction)

1.1 標題

從 0 開始註冊 GitHub 帳號：踏入 GitHub Pages 的第一步

1.2 目標說明

完成本章後，你將能：
	•	建立一個完整可用的 GitHub 帳號
	•	理解 GitHub 與 GitHub Pages 的關係
	•	正確設定基礎安全機制（Email 驗證、2FA）
	•	為後續建立網站專案做好準備

這不是單純註冊帳號，而是為「長期技術資產」建立身份。

1.3 適用人群
	•	完全新手
	•	想用 GitHub Pages 建站但從未接觸 GitHub

約 10–20 分鐘（含安全設定）

第二部分：事前準備 (Preparation)

2.1 材料 / 工具清單
	•	一組可長期使用的 Email（建議使用 Gmail）
	•	可接收簡訊或安裝驗證 App 的手機
	•	穩定網路環境
	•	一台電腦（桌機或筆電）

建議使用電腦操作，手機瀏覽器不利於後續專案管理。

⸻

2.2 必備基礎

不需要會寫程式。
但需要理解：
	•	網站 ≠ Facebook 頁面
	•	GitHub 是程式碼託管平台
	•	GitHub Pages 是靜態網站服務

兩者關係可以這樣理解：

GitHub 是倉庫，GitHub Pages 是把倉庫裡的網站檔案公開展示。

第三部分：核心步驟 (Step-by-Step Instructions)

⸻

3.1 步驟一：進入 GitHub 註冊頁
	1.	打開 https://github.com
	2.	點擊右上角 Sign up
	3.	依序填寫：
	•	Email
	•	Password
	•	Username

Username 建議策略
這裡非常關鍵。

你的 Username 將成為：

https://username.github.io

也就是你未來網站的預設網域。

請避免：
	•	隨便取
	•	帶太多數字
	•	太長
	•	無意義拼湊

如果你想做個人品牌，Username 就是品牌基礎。

⸻

3.2 步驟二：Email 驗證

註冊後會收到驗證信。
請點擊信件中的連結完成驗證。

如果沒有收到信件：
	•	檢查垃圾信件
	•	等 1–2 分鐘
	•	重新發送

⸻

3.3 步驟三：選擇免費方案

GitHub 目前提供 Free 方案即可使用 GitHub Pages。

不需要升級付費。

選擇：

Free Plan → Continue

⸻

3.4 步驟四：完成基本帳號設定

登入後，請立刻做以下設定：

1️⃣ 設定頭像
Profile → Edit profile → 上傳圖片

2️⃣ 填寫個人簡介
這會出現在你的公開頁面

3️⃣ 設定公開 Email（可選）

⸻

3.5 步驟五：啟用兩步驟驗證（強烈建議）

位置：

Settings → Password and authentication → Enable 2FA

建議使用：
	•	Google Authenticator
	•	或 Microsoft Authenticator

這一步非常重要。

因為 GitHub 是你的技術資產庫。
如果被盜，網站與專案都會有風險。

⸻

第四部分：成果驗證 (Verification)

4.1 測試方法

確認：
	•	可以成功登入
	•	Email 已驗證
	•	2FA 已啟用
	•	Profile 頁面可正常顯示

網址：

https://github.com/你的username


⸻

4.2 預期結果

你應該看到：
	•	個人頭像
	•	Username
	•	空白的 Repositories 區域
	•	Profile 頁面可公開瀏覽

這表示你的技術身份已建立完成。

⸻

第五部分：常見問題與故障排除 (Troubleshooting)

⸻

5.1 無法收到驗證信

可能原因：
	•	Email 輸入錯誤
	•	被歸類為垃圾信件
	•	信箱阻擋 GitHub 網域

解法：
	•	重新發送
	•	更換 Email
	•	將 GitHub 加入白名單

⸻

5.2 Username 已被使用

GitHub 是全球平台。

解法：
	•	加入品牌關鍵字
	•	使用簡潔縮寫
	•	使用底線而非亂數數字

⸻

5.3 忘記密碼

使用 Forgot password 重設。

⸻

5.4 2FA 設定失敗

確認：
	•	手機時間同步
	•	驗證 App 正確掃描 QR Code
